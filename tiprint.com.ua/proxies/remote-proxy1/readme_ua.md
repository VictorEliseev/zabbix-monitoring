нструкція з розгортання Zabbix Proxy (Філія)
Цей набір контейнерів забезпечує збір SNMP-трапів від обладнання та їх передачу на центральний сервер через захищений тунель Tailscale.

1. Підготовка системи
Переконайтеся, що на віртуальній машині (Proxmox / Ubuntu) встановлено Docker та Docker Compose.
Виставте права на виконання для допоміжних скриптів, інакше ротація логів та збирач трапів не запустяться:

Bash
chmod +x rotate-tick.sh trap_run.sh
2. Налаштування конфігурації (.env)
Відкрийте файл .env та відредагуйте наступні параметри під конкретну філію:

TS_HOSTNAME: Унікальне ім'я пристрою в Tailscale (наприклад, kyiv-office-lab3).

TS_AUTHKEY: Ключ авторизації з панелі керування Tailscale.

PROXY_HOSTNAME: Ім'я проксі, яке ви створили у веб-інтерфейсі Zabbix.

ZBX_SERVER_IP: IP-адреса центрального Zabbix-сервера всередині мережі Tailscale.

3. Запуск
Виконайте команду в папці з проєктом:

Bash
docker compose up -d
4. Технічні деталі
Автономність: Система сама стежить за вільним місцем на диску.

Ротація: Раз на добу запускається logrotate, який стискає старі логи трапів, якщо вони перевищили 10 МБ.

Безпека: Весь трафік іде через тунель ts-gateway.

Доступ: Порт 162 (SNMP Trap) відкритий тільки локально для прийому даних від обладнання у філії.

Моніторинг: Поточні трапи можна подивитися в реальному часі командою:
tail -f ./snmptraps/snmptraps.log.

5. Пошук несправностей (Troubleshooting)
Перевірка отримання трапів від MikroTik
Якщо в Zabbix не прилітають події, перевірте, чи доходять UDP-пакети до сервера:

Bash
tail -f ./snmptraps/snmptraps.log
Якщо файл порожній, перевірте налаштування SNMP Trap на самому MikroTik (IP має бути адресою сервера, порт 162).

Робота з MIB-файлами
За замовчуванням Zabbix Proxy бачить стандартні OID. Для відображення текстових назв замість цифрових адрес:

Скопіюйте потрібні .mib файли в папку ./mibs на хості.

Перезапустіть контейнер: docker compose restart zbx-proxy-branch.
Проксі підхопить їх автоматично.

Статус тунелю Tailscale
Якщо проксі в статусі "Offline", перевірте з'єднання:

Bash
docker exec -it ts-gateway tailscale status
Переконайтеся, що пристрій авторизований та бачить IP сервера 100.99.7.35.

Перевірка ротації логів
Щоб переконатися, що місце на диску не закінчиться, запустіть перевірку вручну:

Bash
docker exec -it zbx-proxy-branch logrotate -v /etc/logrotate.d/zabbix-traps
