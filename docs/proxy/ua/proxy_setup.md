# **Посібник із налаштування Zabbix Proxy (Active Mode)**

Цей посібник описує розгортання Zabbix Proxy версії 7.0 LTS за допомогою Docker.

## **Конфігурація Docker**

Для стабільної роботи ми використовуємо образ на базі Alpine Linux із вбудованою базою SQLite3.

### **Основні параметри .env:**

* ZBX\_HOSTNAME: Унікальне ім'я проксі в системі Zabbix (наприклад, Office-Lviv-Proxy).  
* ZBX\_SERVER\_HOST: Tailscale IP адреса центрального сервера (наприклад, 100.x.x.x).  
* ZBX\_PROXYMODE: 0 (Active mode — проксі сам підключається до сервера).

## **Інструкція з розгортання**

1. **Налаштування мережі**: Переконайтеся, що Tailscale запущений і сервер доступний за його внутрішньою IP.  
2. **Запуск**:  
   docker-compose up \-d

3. **Перевірка логів**:  
   docker logs \-f zabbix-proxy

## **Реєстрація на сервері (Web UI)**

1. Перейдіть до **Administration \-\> Proxies**.  
2. Натисніть **Create proxy**.  
3. Вкажіть **Proxy name**, який точно збігається з ZBX\_HOSTNAME у вашому файлі .env.  
4. Оберіть **Proxy mode: Active**.  
5. Додайте проксі та зачекайте 1-2 хвилини до появи статусу "Last seen".
