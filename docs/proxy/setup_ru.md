Инструкция по развертыванию Zabbix Proxy (Филиал)
Этот набор контейнеров обеспечивает сбор SNMP-трапов от оборудования и их передачу на центральный сервер через защищенный туннель Tailscale.

1. Подготовка системы
Убедитесь, что на устройстве (Orange Pi / PC) установлен Docker и Docker Compose.
Выставите права на исполнение для вспомогательных скриптов, иначе ротация логов и сборщик трапов не запустятся:

Bash
chmod +x rotate-tick.sh trap_run.sh
2. Настройка конфигурации (.env)
Откройте файл .env и отредактируйте следующие параметры под конкретный филиал:


TS_HOSTNAME: Уникальное имя устройства в Tailscale (например, office-lviv-1).


TS_AUTHKEY: Ключ авторизации из панели управления Tailscale.


PROXY_HOSTNAME: Имя прокси, которое вы создали в веб-интерфейсе Zabbix.


ZBX_SERVER_IP: IP-адрес центрального Zabbix-сервера (внутри сети Tailscale).

3. Запуск
Выполните команду в папке с проектом:

Bash
docker compose up -d
4. Что внутри (технические детали)
Автономность: Система сама следит за местом на диске. Раз в сутки запускается logrotate, который сжимает старые логи трапов, если они превысили 10 МБ.

Безопасность: Весь трафик идет через туннель ts-gateway. Порт 162 (SNMP Trap) открыт только локально для приема данных от оборудования в филиале.

Логи: Текущие трапы можно посмотреть в реальном времени командой:


tail -f ./snmptraps/snmptraps.log.

Поиск неисправностей (Troubleshooting)
1. Проверка получения трапов от MikroTik
Если в Zabbix не прилетают события, проверьте, доходят ли UDP-пакеты до Orange Pi:

Bash
# Просмотр "сырого" лога трапов в реальном времени
tail -f ./snmptraps/snmptraps.log
Если файл пустой, проверьте настройки SNMP Trap на самом MikroTik (IP должен быть адресом Orange Pi, порт 162).

2. Работа с MIB-файлами
По умолчанию Zabbix Proxy видит стандартные OID. Если вы хотите, чтобы в логах вместо цифровых адресов (например, .1.3.6.1.4.1.14988...) отображались текстовые названия (например, mtxrWlRtabInterface), сделайте следующее:

Скопируйте нужные .mib файлы в папку ./mibs на хосте.

Перезапустите контейнеры: docker compose restart zbx-proxy-branch.

Прокси подхватит их автоматически, так как папка примонтирована как readonly.

3. Статус туннеля Tailscale
Если прокси в статусе "Offline" в Zabbix, проверьте соединение:

Bash
docker exec -it ts-gateway tailscale status
Убедитесь, что устройство авторизовано и видит IP сервера 100.99.7.35.

4. Проверка ротации логов
Чтобы убедиться, что логи будут чиститься и место не закончится, можно запустить проверку вручную:

Bash
docker exec -it zbx-proxy-branch logrotate -v /etc/logrotate.d/zabbix-traps
Вы увидите отчет о том, превысил ли файл 10 МБ и нужно ли его сжимать.
