# **Інструкція з розгортання Zabbix Server**

Цей посібник містить вичерпні інструкції для розгортання центрального вузла **Zabbix Server Hub** за допомогою Docker та Tailscale.

## **Склад стеку**

* **Zabbix Server 7.0 (LTS)**: Основне ядро моніторингу.
* **PostgreSQL 16 + TimescaleDB**: База даних, оптимізована для швидкої роботи.
* **init_timescaledb.sql**: Скрипт для активації розширення TimescaleDB.
* **Tailscale**: Захищений VPN-тунель для зв'язку з філіями.
* **Nginx Proxy Manager**: Реверс-проксі для SSL.

## **1. Підготовка середовища**

Перейдіть до директорії сервера та створіть файл конфігурації з шаблону:

```bash
cd server/
cp .env.example .env
nano .env
```

### **Важливо: Авторизація Tailscale**
* **Варіант 1**: Якщо у вас вже є ключ, вставте його в `TS_AUTHKEY`.
* **Варіант 2**: Якщо ключа немає, залиште `TS_AUTHKEY` **пустим**. Після запуску отримайте посилання через логи: `docker logs ts-server`.

### **Ключові параметри (.env):**
* **ZBX_SERVER_NAME**: Відображуване ім'я у веб-інтерфейсі.
* **ZBX_HOSTNAME_SERVER**: Внутрішнє ім'я хоста. **МАЄ СУВОРО ЗБІГАТИСЯ** з полем "Host name" у веб-інтерфейсі Zabbix.

## **2. Розгортання**

Запустіть скрипт автоматизації. Він створить папки та автоматично підключить файл ініціалізації БД:

```bash
chmod +x deploy_server.sh
./deploy_server.sh
```

## **3. Післяінсталяційні налаштування**

### **Налаштування SSL (Nginx Proxy Manager)**
1. Відкрийте веб-інтерфейс NPM (порт 81).
2. Створіть новий **Proxy Host**:
    * **Forward Hostname**: `zabbix-web`
    * **Forward Port**: `8080`
3. Увімкніть **SSL (Let's Encrypt)**.

### **Налаштування самомоніторингу Zabbix**
Щоб статус "ZBX" став зеленим:
1. Перейдіть до **Data collection -> Hosts** та оберіть ваш сервер.
2. **Host name**: Має бути ідентичним параметру `ZBX_HOSTNAME_SERVER` у вашому `.env`.
3. **Interface**: Встановіть тип **DNS**, ім'я `zabbix-agent`, порт `10050`.
4. **Важливо**: Використовується **Zabbix Agent v1** для стабільності в Docker-мережі.

## **4. Обслуговування та діагностика**

* **Перегляд логів**: `docker compose logs -f zabbix-agent`
* **Перезапуск системи**: `docker compose restart`
* **Оновлення конфігурації**: Якщо ви змінили `.env`, виконайте `docker compose up -d` для застосування змін.
