# Інструкція з коректного перенесення документації (Encoding Fix)

Цей метод використовується, якщо при звичайному копіюванні тексту з браузера у VS Code ламається кодування кирилиці (українська/російська мови) або обрізаються довгі ділянки коду.

### Крок 1: Експорт із Gemini
1. У чаті з Gemini натисніть кнопку **«Поділитися» (Share)** над блоком із текстом або в меню документа (Canvas).
2. Виберіть пункт **«Експортувати в Документи» (Export to Docs)**. Система створить тимчасовий Google Doc із вашим текстом.

### Крок 2: Завантаження у форматі Markdown
1. Відкрийте створений Google Документ.
2. Перейдіть у меню **Файл** -> **Завантажити (Download)**.
3. Виберіть формат **Markdown (.md)**. Файл збережеться у вашу системну папку «Завантаження».

### Крок 3: Перенесення в локальний репозиторій
1. Знайдіть завантажений файл у провіднику (Windows Explorer).
2. Натисніть на нього правою кнопкою миші.
3. Виберіть **«Open with Code»** (Відкрити у VS Code).
   * *Це гарантує, що VS Code зчитає файл безпосередньо в кодуванні UTF-8 без втручання буфера обміну.*

### Крок 4: Розміщення в проєкті
1. У відкритому вікні VS Code використайте функцію `Save As...` або просто перейменуйте/перемістіть файл.
2. Помістіть його в потрібну папку згідно зі структурою (наприклад, `docs/encoding_fix_workflow.md`).

---
**Переваги методу:**
* **Збереження кодування:** Google Docs та VS Code коректно працюють з UTF-8 «напряму».
* **Цілісність даних:** Виключається ризик «обрізання» коду, який іноді виникає при копіюванні з веб-інтерфейсів.
